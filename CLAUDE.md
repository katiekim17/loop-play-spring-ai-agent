# CLAUDE.md

## 협업 방식

- 코드를 작성하기 전에 무엇을 왜 하려는지 먼저 설명하고, 사용자가 이해했는지 확인한 뒤 진행한다.
- 여러 단계가 필요한 작업은 단계별로 설명하고 진행 여부를 물어가며 진행한다.
- **PR은 직접 생성하지 않는다.** 커밋과 push까지만 하고, PR 생성은 사용자가 직접 한다. "PR 만들어줘"라고 하면 제목과 본문 문서만 작성해서 제공한다.

## 코드 작성 규칙

- 아키텍처 다이어그램 코드블록에 언어 태그를 명시해야 한다.
- 예외 흐름에서 로그가 누락되지 않도록 try/finally를 사용한다.
- 로깅 시 민감정보(고객 메시지, 주문번호 등) 노출 여부를 점검한다.
- 외부 호출 실패(LLM, API 등)는 @RestControllerAdvice로 잡아 사용자 친화적 메시지와 적절한 HTTP 상태 코드를 반환한다.
- 통계/계산 로직은 컨트롤러 mock 없이 직접 단위 테스트로 분리한다. mock이 고정 응답만 반환하면 다양한 입력에 대한 집계가 검증되지 않는다.
- API 입력값에는 @NotBlank, @Min, @Max 등 Bean Validation 제약을 걸고, 경계값(0, 음수, 상한 초과)과 빈 값에 대한 테스트를 포함한다.
- 테스트 복잡도는 과제 목적에 맞게 조절한다.
